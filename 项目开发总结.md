## 一、项目概述

OrbitDesk 是一个基于 **Qt 6.9.2 (C++)** 的 Windows 桌面效率工具，集成了六大功能模块，采用 **Catppuccin Mocha** 暗色主题，无边框窗口，数据全部持久化到 SQLite。

| 项目属性 | 详情 |
|---|---|
| 技术栈 | C++ / Qt 6.9.2 / CMake / SQLite / Windows API |
| 代码规模 | 20+ 源文件，5000+ 行代码 |
| 数据库 | 8 张表，SQLite |
| 主题 | Catppuccin Mocha 暗色主题 |

## 四、Bug 总结

### 历史 Bug（已修复）

| Bug | 现象 | 根因 | 解决方案 |
|---|---|---|---|
| Bug1: QSS 不生效 | 深色主题未加载 | CMake 自动检测 .qrc 不稳定 | 用 `qt_add_resources()` 显式声明 |
| Bug2: 最大化无法还原 | 点击还原无反应 | showMaximized() 触发 FullScreen 状态 | 保存/恢复 m_normalGeometry |
| Bug3: 最小化冻结 | 恢复后界面无响应 | FramelessWindowHint 丢失 WA_Mapped | changeEvent 中手动恢复 |
| Bug4: 表格无法取消选中 | 点击空白处无反应 | 子 widget 覆盖父 widget 鼠标事件 | `qApp->installEventFilter()` + 坐标转换 |
| Bug5: AI API 认证失败 | 请求返回错误 | HTTP/2 不兼容 + 模型名大小写 | 强制 HTTP/1.1 + 全小写模型名 |
| Bug6: exe 图标不显示 | 程序无图标 | Windows 只支持 .ico 格式 | Python PIL 将 PNG 转 ICO |
| Bug7: 托盘图标不显示 | 关闭后无托盘图标 | 资源路径错误 + 未包含在 qrc | 修正路径 + 添加到 qt_add_resources |
| Bug8: 图标加载失败 | isNull: true | PNG 文件 2.5MB 过大 | 多路径尝试 + 系统默认图标兜底 |
| Bug9: API Key 硬编码 | 安全风险 | 代码中明文显示 | 改为环境变量读取 |
| Bug10: 多实例运行 | 托盘出现多个图标 | 无单实例检测 | QSharedMemory + QLocalSocket |
| Bug11: .gitignore 冲突 | git pull 报错 | 远程和本地有不同 .gitignore | 手动合并，保留本地配置 |

### 今日 Bug（2026.06.07）

| Bug | 现象 | 根因 | 解决方案 |
|---|---|---|---|
| Bug12: 窗口最大化后无法恢复 | 最大化后点击恢复无效 | 未保存原始几何信息 | 新增 m_normalGeometry，最大化前保存，恢复时 setGeometry |
| Bug13: 最大化状态可拖拽 | 全屏状态下拖拽标题栏窗口移动 | eventFilter 未检查窗口状态 | isMaximized()/isFullScreen() 时拒绝拖拽 |
| Bug14: AI 回复显示在错误会话 | 切换会话后回复出现在新会话 | m_messages 共享，切换时被清空 | 重构为 m_sessionMessages（QMap sessionId → QJsonArray） |
| Bug15: "正在思考..."切换后消失 | 切换会话再切回，思考气泡丢失 | switchChatSession 清空布局 | m_pendingSessions 追踪 + 切换时重建气泡 |
| Bug16: 工作流点击太灵敏 | 鼠标按下即触发 | eventFilter 在 MouseButtonPress 中处理 | 改为 MouseButtonRelease 处理 |
| Bug17: 标题栏拖拽后窗口跟随 | 松开鼠标后窗口仍跟随移动 | MouseButtonRelease 处理被工作流 handler 拦截 | 合并 release 处理，优先检查拖拽结束 |
| Bug18: AI 流式输出不显示 | 发送消息后无反应 | 30 秒超时，长文本生成被取消 | setTransferTimeout 从 30s 改为 120s |
| Bug19: 历史趋势图显示 8 小时数据 | 图表时间范围过大 | SQL 查询 datetime('now', '-1 hour') 不准确 | C++ 层额外验证时间戳 + 简化为纯内存收集 |

## 五、技术决策记录

| 决策 | 选择 | 理由 |
|---|---|---|
| UI 框架 | Qt 6 Widgets | 成熟稳定，适合桌面应用 |
| 数据库 | SQLite | 轻量级，无需额外服务 |
| 主题方案 | QSS 全局样式 | 统一管理，支持动态切换 |
| AI 协议 | OpenAI 兼容 | MiMo API 兼容，易于切换模型 |
| 流式输出 | SSE (Server-Sent Events) | 逐字显示，用户体验好 |
| 图标方案 | SVG + QFileIconProvider | 侧边栏用 SVG，工作流用 exe 原生图标 |
| 图表方案 | QPainter 自绘 | 无额外依赖，轻量级 |

## 六、架构设计

```
main.cpp (入口 + 单实例检测)
  │
  └── MainWindow (控制器，持有所有 Manager)
        │
        ├── MonitorPage
        │     ├── SystemMonitor (Windows API)
        │     ├── CircularProgress × 3
        │     └── HistoryChart (QPainter)
        │
        ├── TaskManager → DatabaseManager
        ├── PomodoroManager → QTimer + 信号
        ├── AiChatManager → QNetworkAccessManager
        ├── ChatSessionManager → DatabaseManager
        ├── NoteManager → DatabaseManager
        ├── WorkflowManager → DatabaseManager + QProcess
        ├── QSystemTrayIcon + QMenu
        └── QLocalServer (IPC)
```

**设计模式**：
- 单例模式：DatabaseManager
- 观察者模式：Qt 信号槽
- 状态机：PomodoroManager（4 状态）
- 事件过滤器：eventFilter() 统一处理

## 九、经验总结

### 技术收获
1. Qt 无边框窗口需要手动处理最大化/最小化/拖拽等边界情况
2. SSE 流式输出需要正确处理不完整 JSON chunk 的拼接
3. 多会话并发需要会话级消息隔离，不能共享状态
4. QPainter 自绘图表比引入 Qt Charts 更轻量
5. QFileIconProvider 可以从 exe 提取原生图标

### 调试经验
1. 添加详细 debug 日志追踪信号流和数据流
2. 用终端输出定位问题，比断点调试更高效
3. 复现步骤要精确，才能快速定位 bug

### 架构反思
1. MainWindow 作为"上帝类"承担了过多职责，后续应拆分为 Page Controller
2. SQL 查询散落在 MainWindow 中，应收拢到各 Manager
3. 单线程架构在功能扩展时会遇到瓶颈
