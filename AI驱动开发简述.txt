OrbitDesk 是一个基于 Qt 6 的桌面效率工具，核心解决办公场景中工具碎片化的痛点。用户不再需要在任务管理、笔记、AI 对话、系统监控等多个应用间频繁切换，OrbitDesk 将六大功能整合到一个统一客户端中，通过侧边栏导航一键切换。

整个项目采用 AI Agent 驱动的开发范式：人类开发者负责需求定义和功能验收，AI Agent 负责代码编写、调试和迭代。核心逻辑分三层运作。

第一层是需求到方案的长链推理。用户用自然语言描述需求，Agent 将其拆解为可执行的技术方案。例如"模仿朋友聊天风格"这一模糊需求，Agent 推导出特征提取、Few-shot 示例匹配、RAG 检索三条互补路径的混合方案，并设计出 Parser、Analyzer、FewShotManager、RAGEngine 四个核心类的职责划分。从意图到方案到类设计，层层递进。

第二层是代码实现与闭环迭代。Agent 直接生成完整的 C++ 源码。开发 AI 对话模块时，Agent 需处理 Qt 网络请求、OpenAI 兼容协议、HTTP/1.1 强制降级解决 SSL 问题、环境变量注入、流式信号槽机制等一系列技术细节。开发系统监控模块时，Agent 调用 Windows 原生 API 获取 CPU、内存、磁盘、网络数据，处理首次采样基准值、定时轮询、历史数据清理等底层逻辑。遇到编译错误时，Agent 自主读取报错、定位根因、修复代码，形成"编码、编译、测试、修复"的闭环。

第三层是跨模块协同。各模块共享 SQLite 数据库，通过 Meyer 单例统一管理数据访问。AI 对话与会话管理关联，番茄钟与任务打通，监控数据按时间存储并定期清理。Agent 需维护数据库表结构、信号槽连接、UI 样式在全局范围内的一致性，在多个文件间建立正确的依赖和调用链路。

最终成果：一个涉及系统编程、网络通信、数据库管理、自定义 UI 绘制的完整桌面应用，通过人机协作快速构建完成。开发者定义"要什么"，Agent 实现"怎么做"，验证了 AI 作为核心开发力量参与完整软件工程流程的可行性和高效性。
